User works as a Python-backed ETL developer building and maintaining AWS-based data pipelines and related application-layer services. They primarily use AWS services for ETL workloads, write Python 3.x code for pipeline components and supporting services, and deploy infrastructure using Terraform, AWS SAM, and CloudFormation. They are comfortable with boto3, botocore, pytest, infrastructure-as-code patterns, CI/CD, and AWS-native tooling. Reviews should prioritize ETL correctness, orchestration behavior, IAM/security, deployment safety, performance, and best-practice coverage.
