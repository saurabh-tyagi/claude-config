+## Audit log lifecycle
+
+Audit log entries are written to DynamoDB throughout request processing, keyed
+by `(erasure_id, timestamp_ms)`.
+
+Once a request is successfully processed and reaches the terminal `done` state,
+the process Lambda archives the complete audit log to S3 as a single gzipped
+JSON Lines object at `s3://{AUDIT_LOG_S3_BUCKET}/{AUDIT_LOG_S3_PREFIX}/{id}.jsonl.gz`,
+then deletes the DynamoDB entries for that id.
+
+The `GET /status/{id}` endpoint reads from DynamoDB first and falls back to the
+S3 archive automatically, so reads are seamless before and after archival.
+

+| Variable | Purpose |

+parameters. When running locally, set them in a `.env` file at the project
